    Help on using Petri net plugin

    This plugin is intended for capturing, simulation and verification of Petri net models. Read General help to familiarise yourself with the generic features of Workcraft that are common for all the plugins.

    Create a new work

    In order to create a Petri net work choose File→Create work… menu item. In the New work dialogue box select Petri Net as the model type. You may also want to give the work some meaningful name, e.g. test1, otherwise it will be created with Untitled name. Click OK or press Enter to create the work.

    Capture and edit a model

    A Petri net model can be captured by creating places and transitions and connecting them with the producing and consuming arcs.

    Create elements

    With the place generator , transition generator or text node generator activated you can create a series of places, transition or text notes respectively by clicking the editor panel in the position you want a new element to appear.

    Connect nodes

    When the connection tool is active you can connect places and transitions with arcs by first click the source node and then the destination node. If you hold Ctrl then the destination node becomes a source for the next arc making it easier to form a sequence of connected nodes. Note that places can only be connected to transitions and transitions can only be connected to places. If you attempt to connect two nodes of the same type, then a warning will be issued that connections is not valid.

    Edit the model

    For editing the selection tool must be activated. All the standard features (select, drag-and-drop, delete, copy, undo, group, etc.) work the same. The only new feature is double clicking on a place:

    • Double-click on an empty place to mark it with a token.
    • Double-click a place marked with a token to remove the token.
    • If a place has more than one token then double-clicking does not have any effect.

    Simulation

    For simulation of a Petri net model activete the simulation tool .

    Verification

    A Petri net can be verified for deadlocks via Tools→Verification→Check for deadlocks [MPSat] menu. Custom properties can be verified through formulation a reachability analysis problem in Reach language. This is done a specialised MPSat configuration window accessible via Tools→Verification→Custom properties[MPSat]… menu.
